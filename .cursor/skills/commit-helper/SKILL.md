\# Помощник с коммитами



\## Когда использовать

\- Пользователь пишет: «помоги с коммитом», «сделай коммит», «напиши сообщение для коммита»

\- Пользователь говорит: «используй скилл commit-helper»



\## Инструкция



1\. Выполни `git status --porcelain` — узнай, какие файлы изменены

2\. Выполни `git diff --stat` — пойми масштаб изменений

3\. При необходимости посмотри `git diff` для деталей

4\. Определи тип коммита по Conventional Commits:

&#x20;  - `feat` — новая функция

&#x20;  - `fix` — исправление бага

&#x20;  - `docs` — документация

&#x20;  - `style` — форматирование

&#x20;  - `refactor` — рефакторинг

&#x20;  - `chore` — обслуживание

5\. Напиши сообщение: `тип(область): краткое описание`

6\. Спроси: «Выполнить автоматически или сам сделаешь?»
